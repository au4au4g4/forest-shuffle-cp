# CP 計算方法

本 repo 使用簡化 CP 來比較卡牌或組合技效率。CP 不是官方規則，只是用來快速判斷「這張牌或這組牌值不值得投入」。

---

## 1. 核心公式

```text
總成本 = 費用 + 打出卡牌本身 1 張 + 消耗 1 回合 2 張 − 抽牌 − bonus／額外回合收益
CP = 得分 ÷ 總成本
```

---

## 2. 成本換算表

| 項目 | 成本換算 |
|---|---:|
| 打出該牌本身 | +1 |
| 消耗 1 回合 | +2 |
| 牌面費用 | +費用 |
| 立即抽 1 張 | −1 |
| 立即抽 2 張 | −2 |
| bonus 抽牌 | 依抽牌數扣成本 |
| 額外 1 個完整回合 | 視為抽 2 張，−2 |

---

## 3. 單卡 CP

單卡 CP 適合用來判斷：

```text
這張牌在條件成立時，本身值不值得打？
```

例如大斑啄木鳥：

```text
費用 1
打出本體 +1
回合成本 +2
立即抽 1 張 −1
成功得分 10

成本 = 1 + 1 + 2 − 1 = 3
CP = 10 ÷ 3 = 3.33
```

---

## 4. 整組 CP

整組 CP 適合用來判斷：

```text
整條路線從前置牌到收割牌，平均效率好不好？
```

```text
整組 CP = 整組總分 ÷ 整組總成本
```

整組 CP 會把前置卡牌成本全部算進去，所以通常會比只看收割牌單卡 CP 更低，也更接近實戰。

---

## 5. 最小達標組合

整理套牌時，本 repo 使用「最小達標組合」原則：

```text
只列 CP > 2 的組合。
如果某個小組合已經 CP > 2，就不再列同一路線繼續加牌的版本。
```

例如：

```text
2 大斑啄木鳥成功 + 蒼鷹 已經 CP > 2
```

那就不再列：

```text
2 大斑啄木鳥成功 + 灰林鴞 + 蒼鷹
2 大斑啄木鳥成功 + 松鴉 + 蒼鷹
2 大斑啄木鳥成功 + 紅腹灰雀 + 蒼鷹
```

這樣表格會更像實戰抓牌門檻，而不是列出一堆已經被更小組合支配的版本。

---

## 6. CP 判讀

| CP | 判讀 |
|---:|---|
| < 1 | 偏低，除非為了卡位、開位置或後續 combo |
| 1–1.5 | 可接受，但不是主軸 |
| 1.5–2 | 不錯，值得投入 |
| > 2 | 高 CP，可作為核心路線 |
| > 3 | 爆發型高 CP，通常是條件成熟後的收割牌 |

---

## 7. 注意事項

- CP 是估算，不是絕對強度。
- 有些牌的實戰價值來自補牌、卡位、開位置或阻止對手，CP 不一定完全反映。
- 條件牌要分成「成功 CP」與「失敗 CP」。
- 若牌面規則更新，應先修正卡牌成本與得分，再重算組合 CP。
